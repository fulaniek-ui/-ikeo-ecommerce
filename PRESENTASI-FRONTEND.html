1. Apa Itu IKEO Frontend?

IKEO Frontend adalah website e-commerce furniture untuk customer, dibangun dengan React sebagai Single Page Application (SPA).

🛒 Untuk Customer
Browse produk, keranjang belanja, checkout, pembayaran via Xendit, wishlist, profil, blog
🔗 Terhubung ke Backend
Mengkonsumsi REST API Laravel untuk data produk, autentikasi, order, dan pembayaran

5. State Management (6 Context)

Aplikasi menggunakan React Context API untuk mengelola global state:

ContextFungsiState yang Dikelola
🛍️ ShopContextFilter & sort produksearchQuery, selectedCategory, priceRange, selectedBrands, sortBy, filteredProducts
🛒 CartProviderKeranjang belanjaitems, addItem, updateQuantity, removeItem, emptyCart, totalItems, totalPrice
🌐 LanguageContextMulti-bahasa (ID/EN)language, setLanguage, t() translation function

7. Fitur E-Commerce

🔍 Katalog Produk

  • Filter by kategori (Living Room, Bedroom, dll)
  • Filter by range harga (slider)
  • Filter by brand (multi-select)
Sort
  • Featured (default)
  • Harga: rendah → tinggi
  • Harga: tinggi → rendah
  • Rating tertinggi
  • Nama A-Z

🛒 Shopping Flow

1. Browse produk → filter/search/sort
2. Lihat detail produk → pilih material/size/finish
3. Add to Cart → update quantity
4. Cart page → review items, lihat subtotal + pajak 10%
5. Checkout → isi nama, email, alamat, pilih kurir & metode bayar
6. Place Order → redirect ke Xendit payment page
7. Bayar → redirect ke Payment Status page

8. Multi Bahasa (i18n)

Aplikasi mendukung 2 bahasa menggunakan LanguageContext:

🇮🇩 Bahasa Indonesia (Default)
t("cart.title") → "Keranjang Belanja"
t("products.title") → "Semua Produk"
t("checkout.placeOrder") → "Buat Pesanan"
🇬🇧 English
t("cart.title") → "Shopping Cart"
t("products.title") → "All Products"
t("checkout.placeOrder") → "Place Order"

Semua teks di aplikasi menggunakan fungsi t("key") dari useLanguage() hook. Total 200+ translation keys mencakup navbar, home, products, cart, checkout, profile, blog, about, dan lainnya.

9. Integrasi API Backend

Frontend terhubung ke Laravel backend melalui services/api.ts:

FungsiEndpointKegunaan
fetchProducts()GET /api/productsAmbil produk + filter + pagination
createXenditInvoice()POST /api/xendit/invoiceBuat invoice pembayaran Xendit
getXenditInvoiceStatus()GET /api/xendit/invoice/:idCek status pembayaran

Konfigurasi

API_BASE = import.meta.env.VITE_API_URL || "http://localhost:8000/api"

Base URL API dikonfigurasi via environment variable VITE_API_URL di file .env.

11. Alur Checkout & Payment (Xendit)

1. Customer klik "Lanjut ke Pembayaran" di halaman Cart
2. Halaman Checkout: isi nama, email, alamat pengiriman
3. Pilih kurir: JNE GoSend SiCepat
4. Pilih metode bayar: Transfer Bank E-Wallet
5. Review pesanan: items + subtotal + ongkir + pajak 10%
6. Klik "Buat Pesanan" → createXenditInvoice()
7. Redirect ke halaman pembayaran Xendit
8. Setelah bayar → redirect ke /payment-status
